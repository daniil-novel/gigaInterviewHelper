# Changelog

Все заметные изменения проекта `gigaInterviewHelper` по датам.

## 2026-04-20

### Added
- Добавлен резервный опрос Telegram-диалогов раз в 30 секунд, чтобы после рестарта сервера или сбоя listener приложение подбирало последнее неотвеченное сообщение и продолжало отвечать автоматически.
- Добавлен простой управляющий скрипт `run_app.py` с командами `start`, `status`, `stop` и интерактивным режимом `-i`.
- В интерактивном режиме вопросы и ответы Telegram-бота дублируются в терминал.

### Changed
- Telegram listener теперь не отвечает на запросы оценить диалог и считает их признаком завершённого сценария.
- Bootstrap `/start` в Telegram больше не отправляется, если в чате уже висит входящее неотвеченное сообщение от бота.
- UI и README обновлены: показаны правила паузы после сообщений бота, резервной перепроверки и ручной кнопки остановки автоответчика.

### Fixed
- Устранён сценарий, когда после перезапуска сервера или временного рассинхрона смысловой вопрос в Telegram мог остаться без ответа.
- Устранён риск лишнего ответа на финальные системные сообщения и просьбы оценить GigaRecruiter.
- Исправлен импорт нового PDF-резюме через OpenRouter: резюме теперь парсится нативной PDF-подачей в `google/gemini-3.1-pro-preview`, а выбираемая в настройках модель используется только для ответов.
- Исправлен сценарий, когда первое новое сообщение от GigaRecruiter после завершённой сессии не получало ответ: теперь чат проверяется каждые 30 секунд даже без активной сессии, а подходящая сессия автоматически перепривязывается по позиции из текста сообщения.

## 2026-04-05

### Added
- Добавлен полноценный Gmail OAuth 2.0 flow с `authorization code`, `refresh_token` и чтением писем через Gmail API.
- Добавлена диагностика Gmail OAuth и понятные сообщения об ошибках Google API с прямой ссылкой на включение Gmail API.
- Добавлен автоматический и ручной запуск сессии из входящих писем через кнопку `Начать сессию`.
- Добавлен блок статистики по авто-режиму: приглашения, авто-сессии, вопросы, подтверждённые ответы и текущая очередь.
- Добавлен `CHANGELOG.md` для фиксации изменений по дням.

### Changed
- Основной пользовательский поток упрощён до сценария: письмо -> сессия -> открыть AI-интервью в браузере -> вставить ответ вручную.
- Интерфейс переведён на тёмную визуальную систему с более сильным контрастом и адаптивной сеткой.
- Дашборд перестроен в табовый layout: `Сессии`, `Входящие письма`, `Профиль`, `Настройки`.
- Навигация после действий стала контекстной: сохранения и проверки возвращают пользователя в соответствующую вкладку.
- Продвинутая Telegram-интеграция понижена до optional advanced-режима и убрана из основного сценария.

### Fixed
- Исправлен баг с отсутствующим импортом `parsedate_to_datetime` в Gmail-сервисе.
- Исправлены raw JSON-ответы в браузере: Gmail-проверки и почтовая синхронизация теперь показывают понятные UI-уведомления.
- Улучшен парсинг ссылок из HTML-писем: отбрасываются технические URL вроде `w3.org`, а приоритет получают реальные interview/invite/chat ссылки.
- Исправлены проблемы адаптивности интерфейса, из-за которых action-кнопки и боковые блоки ломали layout на плотных экранах.
- Исправлен разъезжающийся блок действий в карточке ответа: кнопки подтверждения, перегенерации, редактирования и пропуска теперь стабильно укладываются в адаптивную сетку без наложения текста.

### Repository
- Подготовлен `.gitignore` для безопасной публикации проекта.
- Проект оформлен и опубликован в GitHub-репозиторий `gigaInterviewHelper`.
